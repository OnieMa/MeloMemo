# MeloMemo 手动测试清单

本文档记录当前这批改动包含的功能，以及回家后建议如何逐项测试。

## 测试前准备

1. 拉取代码后先安装依赖并启动前后端。
2. 如果使用已有 SQLite 数据库，后端启动时会自动给 `UserWordLookup` 补充 `sourceSongRefs` 字段；也可以额外执行一次 `npm run db:push` 让 Prisma schema 和数据库保持一致。
3. 建议准备 2 首带歌词的本地歌曲，用于测试“同一个单词出现在多首歌中”的场景。
4. 测试时打开浏览器 DevTools 的 Network 面板，方便确认本地模式不会写入后端，登录模式会写入后端。

## 1. 打开 App 后的入口模式

功能说明：

- App 打开后默认进入“我的”页面。
- 未登录时不会自动进入本地用户状态。
- 用户可以明确选择“本地使用模式”，或选择登录/注册进入云端模式。

测试步骤：

1. 清空浏览器本地存储，刷新页面。
2. 确认首屏是“我的”页面。
3. 确认页面上能看到本地使用入口和登录入口。
4. 点击“本地使用模式”，确认进入本地模式并能使用歌曲库等功能。
5. 退出登录或清空 token 后刷新，确认不会自动把用户当成本地用户登录。

预期结果：

- 未选择本地模式前，不应自动加载本地用户。
- 选择本地模式后，后续数据保存在本机 IndexedDB。

## 2. 本地模式和登录模式的数据边界

功能说明：

- 本地模式下，歌曲、生词、收藏、查词记录、学习时长都只保存在本机。
- 登录模式下，后续操作写入后端。
- 从本地模式切换到登录模式时，会把已有本地数据同步到后端。

测试步骤：

1. 进入本地模式。
2. 上传或导入至少 1 首本地歌曲。
3. 收藏歌曲。
4. 播放歌曲并点击几个歌词单词查询。
5. 把其中几个单词加入生词本。
6. 观察 Network 面板，本地模式下不应出现保存歌曲、生词、收藏、查词记录到后端的请求。
7. 使用邮箱登录、注册，或微信登录。
8. 登录成功后，确认云端歌曲库、生词本、收藏、重点词汇、学习时长都出现本地数据。
9. 再次刷新并登录同一账号，确认数据不会重复膨胀。

预期结果：

- 本地模式不会把数据写入服务器。
- 登录成功后，本地歌曲会上传到后端。
- 本地收藏、生词、查词次数、单词来源歌曲、学习时长会同步到后端。
- 重复登录不会重复创建同一首本地同步歌曲；查词次数和学习时长采用合并策略，不应每次登录都翻倍。

注意事项：

- 同步不会删除本地数据。
- 如果登录成功但同步失败，“我的”页面会显示类似“登录成功，但本地数据同步失败”的错误信息。

## 3. 本地数据同步到后端

功能说明：

- 本地歌曲同步到 `/api/songs`，使用 `sourceType=local-sync` 和本地歌曲 `id` 去重。
- 本地收藏同步到 `/api/favorites`。
- 本地生词同步到 `/api/vocabulary`。
- 本地查词记录同步到 `/api/word-lookups/sync`。
- 本地学习记录同步到 `/api/study/sync`。

测试步骤：

1. 本地模式下创建数据：
   - 上传 2 首歌曲。
   - 每首歌至少点击 2 个单词。
   - 同一个单词尽量在两首歌里都点击一次。
   - 收藏至少 1 首歌。
   - 把至少 2 个单词加入生词本。
   - 播放歌曲 1 分钟以上。
2. 登录账号。
3. 在 Network 面板确认出现歌曲上传、收藏、生词、查词同步、学习记录同步请求。
4. 进入云端模式后刷新页面。
5. 检查云端歌曲库是否有刚才的本地歌曲。
6. 检查收藏是否保留。
7. 检查生词本是否保留。
8. 检查重点词汇中的查词次数是否保留。
9. 检查进度页学习时长是否保留。

预期结果：

- 所有本地数据都能在登录后的云端模式看到。
- 本地歌曲再次同步时应复用已有云端歌曲，不重复上传。
- 生词和收藏不会重复出现。

## 4. 单词查词次数和歌曲来源统计

功能说明：

- 每次点击歌词单词都会记录查词次数。
- 每个单词会记录它被学习过的歌曲列表。
- 同一个单词在同一首歌内重复点击，会增加该歌曲内的次数。
- 同一个单词在不同歌曲中点击，会增加“学习过的歌曲数”。

测试步骤：

1. 在歌曲 A 中点击单词 `love` 3 次。
2. 在歌曲 B 中点击单词 `love` 2 次。
3. 进入词汇/单词本相关页面。
4. 查看重点词汇卡片中的 `love`。
5. 查看生词本按单词展示中的 `love`。

预期结果：

- `love` 的总查词次数应增加。
- `love` 应显示学习过 2 首歌。
- 单词卡片中歌曲标签应显示每首歌内的查询次数。

## 5. 生词本两种展示模式

功能说明：

- 生词本支持“按歌曲”和“按单词”两种展示模式。
- 默认展示为“按歌曲”。
- 两种模式都支持“按添加时间”和“按名称”排序。
- 按歌曲模式下，点击歌曲行后展开该歌曲下的所有生词。
- 按单词模式下，每个单词会显示学习过哪些歌曲。

测试步骤：

1. 先在不同歌曲里收藏多个生词。
2. 进入“我的生词本”区域。
3. 确认默认是“按歌曲”。
4. 点击一首歌曲，确认展开后能看到该歌曲下所有生词。
5. 切换排序为“名称”，确认歌曲列表或单词列表按名称排序。
6. 切换排序为“添加时间”，确认最新学习/收藏的内容靠前。
7. 切换到“按单词”。
8. 检查每个单词卡片是否显示“学习过 N 首歌”。
9. 点击可跳转的歌曲标签或“跳转到歌词位置”，确认能跳到对应歌曲和时间点。

预期结果：

- 两种展示模式都能正常切换。
- 排序按钮对当前模式生效。
- 按歌曲能查到歌曲下的所有生词。
- 按单词能查到该单词学习过的歌曲。

## 6. 点击单词发音时的背景音乐策略

功能说明：

- 设置页新增“单词发音背景”选项。
- 默认策略是“压低”背景音乐。
- 可切换为“暂停”背景音乐。
- 单词发音结束后，背景音乐会恢复音量或继续播放。

测试步骤：

1. 播放一首歌曲。
2. 保持默认“压低”设置。
3. 点击歌词中的单词，触发单词发音。
4. 确认歌曲没有停止，只是音量降低。
5. 进入“我的”页面的播放设置，把策略改为“暂停”。
6. 回到播放页，继续播放歌曲。
7. 点击单词发音。
8. 确认歌曲暂停，单词发音结束后歌曲继续播放。

预期结果：

- “压低”模式下，背景音乐继续播放但音量明显降低。
- “暂停”模式下，背景音乐暂停，单词发音结束后恢复播放。
- 设置刷新后仍保持用户选择。

## 7. 歌词滚动时显示已查词注释

功能说明：

- 设置页新增“已查词注释”开关。
- 开启后，歌词滚动时会在已查询过的单词上方显示音标/释义摘要。
- 注释记录保存在本地浏览器，用于反复记忆。

测试步骤：

1. 进入播放页。
2. 点击几处歌词单词并完成查询。
3. 进入“我的”页面，打开“已查词注释”。
4. 回到播放页播放歌曲。
5. 观察当前歌词行和附近歌词中已查询过的单词。
6. 关闭“已查词注释”，再次播放观察。

预期结果：

- 开启后，已查询过的单词上方显示简短注释。
- 已注释单词有视觉标记。
- 关闭后，不再显示注释。

## 8. 生词本和歌词注释的关系

功能说明：

- 点击歌词单词会记录查词历史和歌词注释。
- 点击“加入生词本”会把该单词加入生词本。
- 生词本按歌曲聚合时，会结合查词来源歌曲和收藏来源歌曲。

测试步骤：

1. 点击某个歌词单词，只查询但不加入生词本。
2. 确认它会出现在重点词汇统计中。
3. 再点击“加入生词本”。
4. 确认它出现在生词本中。
5. 在另一首歌中查询同一个单词。
6. 回到生词本按单词查看。

预期结果：

- 查询过但未收藏的单词参与查词统计。
- 加入生词本后才出现在“我的生词本”。
- 同一个生词可以显示学习过多首歌。

## 9. 退出登录后的行为

功能说明：

- 退出登录后回到“我的”页面。
- 不会自动进入本地模式。
- 用户需要重新明确选择本地模式或登录。

测试步骤：

1. 登录账号。
2. 确认云端数据加载正常。
3. 退出登录。
4. 刷新页面。

预期结果：

- 页面回到“我的”。
- 不自动显示本地用户状态。
- 可以重新选择本地模式或登录。

## 10. 回归测试重点

建议重点确认这些旧功能没有被影响：

- 本地歌曲上传和播放。
- YouTube 搜索/下载。
- 歌词点击查词。
- 生词保存。
- 收藏歌曲。
- 歌词编辑保存。
- 播放列表顺序播放、随机播放、循环播放。
- 讯飞 TTS 发音。
- 原唱片段慢放。

## 已知注意点

- 旧版本已经产生的查词记录没有完整歌曲来源时，只能通过生词收藏里的来源歌曲做兜底。
- 本地同步到云端后，本地数据仍保留，不会自动清空。
- 文档中的“按歌曲/按单词”生词本区域位于当前词汇掌握/发现视图里的“我的生词本”。
- 本次只做代码和文档改动，没有启动项目，也没有执行构建。
