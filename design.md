> 设计稿来源: Stitch 项目 "Melo'Memo" (projects/14557199556707371076)

> 最后更新: 2026-04-14

## 1. 创意北极星: "The Sonic Sanctuary"

MeloMemo 的设计理念是将 App 从「工具」升华为「乐器」。界面不是一套模板，而是一个沉浸式的高端学习体验，像音乐本身一样流畅、充满情感。

核心打破模板的三个手段：
- **有意识的不对称**：让歌词流 (LyricStream) 以不同行长和有机布局自由呼吸
- **大气层次感**：使用玻璃拟态 (Glassmorphism) 使界面像漂浮在生动音景之上
- **字体灵魂**：将 *Plus Jakarta Sans* 的几何权威感与 *Be Vietnam Pro* 的亲和温暖感混搭

## 2. 设计系统

### 2.1 色彩系统

#### 主要颜色
| Token | 值 | 用途 |
|---|---|---|
| `primary` | `#5543cf` | 主要交互、CTA、渐变起点 |
| `primary-container` | `#9e93ff` | 渐变终点、轻量强调 |
| `secondary` | `#006764` | 次要交互、进度条渐变起点 |
| `secondary-container` | `#5af9f3` | 进度条渐变终点、辅助亮点 |
| `tertiary` | `#a1305e` | 第三层强调（标签、状态） |
| `tertiary-container` | `#ff8db3` | 第三层容器 |

#### 表面层级（Tonal Layering）
| Token | 值 | 层级含义 |
|---|---|---|
| `surface` | `#fbf4ff` | 基础底层 |
| `surface-container-low` | `#f6edff` | 内容区块（一级） |
| `surface-container` | `#efe3ff` | 卡片容器 |
| `surface-container-high` | `#eaddff` | 交互容器 |
| `surface-container-highest` | `#e5d6ff` | 活跃交互卡片 |
| `surface-container-lowest` | `#ffffff` | 最高层（玻璃基底） |
| `surface-dim` | `#ddccff` | 禁用/淡化状态 |

#### 文字颜色
| Token | 值 | 用途 |
|---|---|---|
| `on-surface` | `#34284f` | 主要文字（禁用纯黑） |
| `on-surface-variant` | `#62557f` | 次要文字、元数据 |
| `outline` | `#7d709b` | 轮廓线（谨慎使用） |
| `outline-variant` | `#b5a6d5` | 鬼影边框（只用 15% 不透明度） |

#### "无线"原则
**禁止使用 1px 实线边框进行分区。** 区块边界只通过色调差异定义。若需要分隔内容，使用 32px 垂直间距或从 `surface` 到 `surface-container-low` 的微妙色调切换。

### 2.2 字体系统

| 角色 | 字体 | 特点 |
|---|---|---|
| Display / Headline | **Plus Jakarta Sans** | 高对比、粗体、编辑风格 |
| Body / Label | **Be Vietnam Pro** | 优化快速阅读的可读性 |

字体层级规则：
- `display-lg`：歌曲标题，营造「杂志封面」冲击感
- `headline-sm`：SmartWord 单词释义，权威感融入学习时刻
- `title-md`：SmartWord 静止状态标注

### 2.3 圆角系统 (Roundness: FULL)

| 级别 | 值 | 适用 |
|---|---|---|
| `sm` | `0.5rem` | 最小允许值 |
| `DEFAULT` | `1rem` | 常规组件基准 |
| `lg` | `2rem` | 大卡片、EchoFeedback 容器 |
| `xl` | `3rem` | GlassControl 播放栏、分节间距 |
| `full` | `9999px` | 按钮、Chip（强调「Lively」品牌） |

### 2.4 间距系统 (Spacing Scale: 3)

| 场景 | 值 |
|---|---|
| 组件内部 | `1rem` (DEFAULT) |
| 区块间距 | `3rem` (xl) |
| 触控目标最小尺寸 | `48×48dp` |

### 2.5 阴影与深度

- **禁止**使用传统重阴影
- **环境阴影 (Ambient Shadow)**：32px blur、0px offset、6% opacity，颜色取自 `on-surface` (#34284f)
- **玻璃拟态**：`surface-container-lowest` 60% 透明度 + `backdrop-blur: 20-24px`
- **鬼影边框**：仅在高密度区域使用 `outline-variant` (#b5a6d5) 15% 不透明度

## 3. 特征组件

### 3.1 LyricStream（歌词流 / 主动学习模式）
- **激活状态**：`display-sm` + `on-surface`，高对比，轻微放大 1.05x
- **非激活状态**：`display-sm` + `outline`，应用 `blur(4px)` 模拟周边视野聚焦
- **段落间距**：`xl` (3rem) 垂直节奏

### 3.2 SmartWord（互动词汇）
- **静止**：`title-md` + `primary-container` 下划线（3px，圆角）
- **交互**：点击触发 `surface-container-highest` 弹出框
- **弹出框规则**：无边框，使用环境阴影与歌词背景分离；允许弹出框轻微叠压上方歌词行以营造深度感

### 3.3 GlassControl（播放引擎控制栏）
- **材质**：60% `surface-container-lowest` + `backdrop-blur: 24px`
- **形状**：`xl` (3rem) 圆角，药丸形「悬浮」美学
- **进度条**：从 `secondary` (#006764) 到 `secondary-container` (#5af9f3) 的渐变

### 3.4 EchoFeedback（音频录制）
- **波形**：动态高度，用户声音用 `primary`，参考音轨用 `secondary`
- **容器**：`surface-container-low` + `lg` (2rem) 圆角

## 6. 设计规范 Do's & Don'ts

### Do
- 使用叠压元素，例如 SmartWord 弹出框轻微叠压上方歌词行，营造深度
- 按钮和 Chip 使用 `full` (9999px) 圆角，强调「Lively」品牌支柱
- 录制状态使用 `primary-fixed-dim` 提供温暖、不刺眼的光晕

### Don't
- 不使用纯黑 (#000000) 文字，使用 `on-surface` (#34284f)
- 不使用 Material Design 标准分割线，用 32px 垂直间距或 `surface` → `surface-container-low` 色调过渡
- 不使用 90° 直角，最小圆角 `sm` (0.5rem)，首选 `DEFAULT` (1rem)
- 不使用重阴影，使用带颜色的环境阴影（6% opacity）
- 不堆积控件，非即时任务的控件隐藏于「更多」菜单或降至 `on-surface-variant` 低透明度

## 7. 技术实现参考

- **图标库**: Material Symbols (Google Fonts)
- **CSS 效果**: `backdrop-filter: blur(20-24px)` 玻璃拟态
- **CTA 渐变**: `linear-gradient(#5543cf, #9e93ff)`（primary → primary-container）
- **进度渐变**: `linear-gradient(#006764, #5af9f3)`（secondary → secondary-container）
- **环境阴影**: `box-shadow: 0 0 32px 0 rgba(52, 40, 79, 0.06)`
- **色调系统**: Material Design 3 色彩 token 体系
